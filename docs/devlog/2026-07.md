# 2026-07 开发日志

## 2026-07-01

### 火山方舟 TTS 全链路修复（1.0 + 2.0）

**背景**：`Doubao-tts2.0` 调用报 `resource ID is mismatched with speaker related resource`，只有 1.0 能通。用户反馈"2.0 音色应该是参数，不需要单独订阅"，触发深入调查。

**调查结论**（用 browse 工具核实火山官方三份文档）：

- 音色列表页（`/docs/6561/1257544`）：2.0 音色是公开参数 `voice_type`，**无需订阅**，直接传即可。后缀是 `uranus`/`mars`（不是早期笔记里的 jupiter/saturn）。
- 单向流式 HTTP API 参考（`/docs/6561/2528925`）：端点 `https://openspeech.bytedance.com/api/v3/tts/unidirectional`，认证 `X-Api-Key` + `X-Api-Resource-Id: seed-tts-2.0` + `speaker`。与现有代码完全一致。
- 错误码页（`/docs/6561/2534853`）：`resource ID is mismatched` 的官方排查清单是「服务未开通 / 音色 ID 拼写错 / 复刻过期 / 音色无权限」。
- **直连火山验证**：`curl` 用渠道 Key + `seed-tts-2.0` + `zh_female_vv_uranus_bigtts` 成功返回 33KB MP3。证明火山侧、Key 权限、服务开通、v3 协议全都没问题，**根因在平台代码**。

**根因（三层）**：

1. **API 层**：`relay/channel/volcengine/tts.go` 的 `openAIToVolcengineVoiceMap` 把 `alloy` 绑到 1.0 音色 `zh_male_M392_conversation_wvae_bigtts`，其余 5 个别名是 2.0 音色。调 `Doubao-tts2.0` + `voice:"alloy"` → 2.0 resource + 1.0 音色 = mismatch。文档示例全用 `voice:"alloy"`，雪上加霜。
2. **操练场层**：`Playground/index.jsx` 把 audio 模型当视频任务发到 `/pg/video/generations` → `RelayTask` → `taskdoubao.TaskAdaptor`（视频 adaptor，不处理 TTS），且 `taskPayload` 没传 `voice`、用 `prompt` 而非 `input`。操练场调 TTS 必然失败。`helpers/api.js` 还把 audio 模型从操练场下拉过滤掉了。
3. **计费层**：`doubao-tts`/`doubao-tts2.0` 既无 ModelRatio 也无 ModelPrice，走默认 37.5 惩罚倍率。

**计费方案核实**：系统原生支持按次计费（`defaultModelPrice` + `calculateTextQuotaSummary`/`calculateAudioQuota` 的 `UsePrice` 分支 = `ModelPrice * QuotaPerUnit * GroupRatio`，不看 token）。Volcengine TTS 的 `DoResponse` 虽 `CompletionTokens=0`、走 `PostTextConsumeQuota`，但 `calculateTextQuotaSummary` 的 UsePrice 分支同样按次计费，且 `PromptTokens>0` 不触发「total tokens 0 不扣费」分支。**不需要改 DoResponse**。

**改动**（方案 1：voice 透传原生音色 ID + 别名按版本兜底）：

**验证中发现并修复的额外问题**：

1. **音色后缀与模型版本对应关系**（与最初假设相反）：直连火山批量测试发现 `uranus` 后缀音色属于 `seed-tts-2.0`，`mars` 后缀和 M392 旧格式属于 `seed-tts-1.0`。最初把 mars 音色（cancan/wenhao/amanda 等）错放进 2.0 列表，实测报 mismatch 后全部修正：2.0 列表只用 uranus（12 个多语种），1.0 列表用 mars + M392（12 个）。V2 别名映射表也改成全 uranus。规律写进代码注释和文档 Callout。

2. **操练场音色下拉不显示**：后端 `GetModelSupportEndpointTypes` 给 TTS 模型返回 endpoint 是 `openai`（非 `audio-speech`），前端 `isAudioModel` 判断 `ep==='audio-speech'` 永远不成立。改成按模型名判断（`isAudioSpeechModel`，大小写不敏感），SettingsPanel 和 index.jsx 都用此函数。

3. **音频链接点击跳空页面**：react-markdown 10 的 `defaultUrlTransform` 用协议白名单 `https?|ircs?|mailto|xmpp`，把 `data:` 和 `blob:` 协议的 href 清空，导致 `<a href="">` 点击导航到当前页。修复：MarkdownRenderer 传自定义 `urlTransform`，对 `data:audio` 和 `blob:` 透传。同时 sendAudioRequest 改用 base64 data URL（不依赖页面生命周期）。

4. **音频播放器过小**：`<audio>` 默认宽度不足，给 figure 加 `maxWidth:520px`、audio 加 `minWidth:280px`。
