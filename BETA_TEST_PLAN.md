# Vancine 封闭测试计划

## 一、测试目标

- 验证产品核心流程是否顺畅
- 收集真实用户反馈
- 发现并修复阻塞性问题
- 建立首批种子用户

## 二、测试用户筛选标准

### 理想测试用户

| 特征 | 说明 |
|------|------|
| 身份 | 独立开发者或小型团队技术负责人 |
| 需求 | 正在使用或计划使用 AI API |
| 意愿 | 愿意花 15-30 分钟测试并反馈 |
| 能力 | 能独立完成 API 接入 |

### 不适合的用户

- 没有技术背景
- 不愿意提供反馈
- 只想获取免费额度

## 三、邀请渠道

### 1. 个人网络

- 技术圈朋友
- 前同事
- 开发者社群好友

**邀请话术**：
> 我在做一个 AI 模型聚合平台，支持调用中国的图片、视频、3D 模型。想邀请你帮忙测试一下，会给一些免费额度。只需要 15 分钟，测试一下 API 接入流程，然后告诉我体验如何。

### 2. 开发者社区

- V2EX 节点「酷工作」或「分享创造」
- GitHub Discussions
- Discord/Telegram 开发者群

**邀请帖标题**：
> [封闭测试] 寻找 5 名开发者测试 AI 模型聚合平台

**邀请帖内容**：
> 我在做一个 AI 模型聚合平台，主要提供中国图片、视频、3D 模型的统一 API。
>
> 现在寻找 5 名开发者帮忙测试：
> - 测试内容：完成一次 API 调用（文本/图片/视频任选）
> - 预计时间：15-30 分钟
> - 回报：$5 免费额度 + 15 分钟一对一答疑
>
> 感兴趣请私信我，附上你的技术背景和使用场景。

### 3. 精准邀请

在 Reddit、Hacker News 等平台找到正在讨论 AI API 的用户，私信邀请。

## 四、测试流程

### 第一步：邀请（D1-D2）

1. 发布邀请帖/私信
2. 筛选合适的用户
3. 发送测试账号和 API Key

### 第二步：测试（D3-D5）

用户需要完成的任务：

| 任务 | 优先级 | 预计时间 |
|------|--------|----------|
| 注册账号 | 必选 | 2 分钟 |
| 获取 API Key | 必选 | 1 分钟 |
| 完成第一次 API 调用 | 必选 | 10 分钟 |
| 测试不同模型（至少 2 个） | 推荐 | 5 分钟 |
| 查看使用日志 | 推荐 | 2 分钟 |

### 第三步：反馈（D5-D7）

收集反馈的方式：

1. **问卷**（推荐）— 使用 Google Forms 或腾讯问卷
2. **访谈** — 15 分钟视频/语音通话
3. **文字反馈** — 邮件或社区私信
